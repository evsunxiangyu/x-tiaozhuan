# X 新标签页打开

Chrome 扩展：在 X (Twitter) 中点击帖子或评论时，在新标签页打开而不是当前页跳转。

## 功能

- **Feed 点击新标签打开** — 在首页时间线点击任意推文，自动在新标签页打开，不离开当前页
- **详情页原贴不跳转** — 在推文详情页点击原贴/父推文仅阻止跳转，不重复开标签
- **图片放大正常工作** — 点击推文中的图片正常放大浏览，不受拦截影响
- **外链正常跳转** — 推文中引用的外部链接（飞书文档等）点击正常打开
- **交互按钮不受影响** — 回复、转发、点赞、收藏等按钮正常使用
- **一键开关** — 点击扩展图标即可启用/关闭，蓝色图标=启用，灰色+OFF badge=关闭

## 安装

1. 下载或克隆本项目
2. 打开 Chrome，访问 `chrome://extensions/`
3. 开启右上角「开发者模式」
4. 点击「加载已解压的扩展程序」，选择本项目文件夹

## 技术实现

- **capture 阶段拦截** — 在 React 事件处理之前捕获点击事件，阻止 SPA 导航
- **精准选择器判断** — 通过 `data-testid` 选择器区分图片、链接、嵌套推文、交互按钮等不同元素
- **DOM 位置检测** — 利用 `compareDocumentPosition` 判断详情页中的原贴与主推文关系
- **OffscreenCanvas 图标** — 动态生成扩展图标，无需打包图片资源
- **MV3 Service Worker** — 使用 Manifest V3 标准的 background service worker

## 处理流程

```
点击推文 → 检查开关状态
  ├─ 图片 → 放行（X 原生放大）
  ├─ 外部链接 → 放行
  ├─ 嵌套推文 / 引用卡片 → 阻止跳转
  ├─ 交互按钮 → 放行
  ├─ 详情页原贴 → 仅阻止跳转
  └─ Feed 推文 → 阻止跳转 + 新标签页打开
```

## 文件结构

```
├── manifest.json   # 扩展配置（MV3）
├── background.js   # 开关状态管理 + 动态图标生成
└── content.js      # 点击拦截核心逻辑
```
